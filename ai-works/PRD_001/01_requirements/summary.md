# 阶段 01 摘要 - 需求澄清

> PRD_001 | 可视化模块

## 核心结论

实现 Dashboard 可视化模块，为用户提供**快速查看 + 深度分析**的双模式体验。采用赛博朋克风格，数据驱动，激励导向。

## 关键产出

### Must Have 功能清单

| 组件 | 功能 |
|------|------|
| 概览卡片组 | 今日统计、目标达成率、连续天数、周对比 |
| 行为趋势图 | 折线图 + 类型筛选 + 时间范围 |
| 活跃度热力图 | GitHub 风格日历，12个月视图 |
| 目标进度环 | 环形进度条，点击可钻取 |
| 时间选择器 | 今日/7天/30天/90天/自定义 |

### Nice to Have

- 分类饼图、柱状对比图
- 数据导出 (PNG/CSV)
- 迷你 Sparkline

## 供后续阶段使用

### 02-PRD 需要
- 功能范围定义 (Must/Nice/不做)
- 用户角色定义
- 成功指标

### 03-UI设计 需要
- 四类用户旅程
- 组件清单
- 响应式要求

### 04-技术设计 需要
- 数据依赖 (behavior_records, goals, notes)
- 性能边界 (首屏<2s, 图表<500ms)
- 数据量假设 (<10000条/年)

## 注意事项

1. **空状态处理** - 无数据时需引导用户开始记录
2. **大数据量** - 热力图限12个月，趋势图需聚合
3. **移动端** - 单列布局，图表支持横向滚动
4. **图表库选型** - 需支持深色主题 + 响应式
