# 关键细节

1. 使用非对称加密处理敏感信息比如密码
2. client side 组件禁止操作数据库，禁止泄漏敏感环境变量到 client side，应该通过定义和调用 api 来处理
3. 优先使用开源库
4. 优先使用自动化而不是硬编码，比如 sql 文件的生成应该通过 entity 的自动化
5. 应该使用缓存，比如前端使用 tanstack useQuery 和 useMutation 去和 remote 交互
6. 完善的错误处理
7. 完善的 loading 和 pending 处理
8. 使用 chakra 的 theme 来配置样式，尽量不要自定义 class

# 实现流程

每一步实现完后，需要停下来等待人类 review

1. 按模块拆分成独立过程
2. 先处理 entity 的新建或修改，并处理数据库
3. 接着更新后端对应的数据类型
4. 然后更新后端纯函数
5. 更新后端服务和 api
6. 接着处理前端部分，更新前端类型定义
7. 更新前端纯函数
8. 更新前端服务/store/hook
9. 更新 theme 系统
10. 更新前端组件
11. 更新前端页面集成
