# 继续工作流

从快照恢复，继续暂停的工作流。

## 参数

$ARGUMENTS

格式: `[单号]?`（可选，不提供则继续最近的工作流）

## 执行步骤

1. 确定要继续的工作流：
   - 如果提供了单号，使用该单号
   - 否则，找到最近暂停/进行中的工作流
2. 读取 `overview.json`
3. 查找最新快照
4. 恢复 Context 缓存
5. 切换 Git 分支
6. 处理未提交变更（如有）
7. 从暂停点继续执行

## 恢复内容

- 工作流状态
- 当前阶段进度
- Context 缓存（project.md, tech-config.md, ui-config.md）
- 阶段产出摘要

## 快照类型

| 类型 | 说明 |
|------|------|
| `stage_complete` | 阶段完成时自动保存 |
| `manual` | 用户手动保存 |
| `auto_pause` | 长时间暂停自动保存 |
| `before_rework` | 返工前自动保存 |

请执行恢复操作。
