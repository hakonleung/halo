# 工作流学习指令（已合并到 archive 指令）

> **注意**：此指令已合并到 `/flow-archive` 指令中。归档工作流时会自动执行学习流程，无需单独执行此指令。

从归档的工作流中提取技术设计和技术经验，更新到项目配置文件中，确保后续 AI flow 可以吸取教训。

## 触发指令

```
/flow-learn
```

**已废弃**：请使用 `/flow-archive` 指令，归档时会自动执行学习流程。

## 重要提示

**在执行此命令前，必须先读取完整的详细说明文件：**

`ai-coding/action-learn.md`

该文件包含完整的执行流程、详细步骤、输出格式等所有必要信息。请先读取该文件，然后按照其中的详细说明执行。

## 执行流程

1. 查找所有归档的 works，过滤未标记 learned 的归档
2. 读取 archive.md 和 overview.json
3. 提取技术设计和技术经验
4. 简化并分类内容
5. 更新 context/tech-config.md、.cursorrules、CLAUDE.md
6. 标记归档为 learned
7. 提交到 Git

## 参数说明

无需参数。

## 适用场景

**已废弃**：此指令已合并到 `/flow-archive` 指令中。

如果您需要单独学习已归档的工作流（不重新归档），可以：
1. 手动读取 `ai-works/_archived/[单号]/archive.md`
2. 手动提取技术经验
3. 手动更新配置文件

但建议直接使用 `/flow-archive` 指令，归档时会自动执行学习流程。
