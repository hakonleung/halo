# 轻量工作流指令

快速迭代模式，跳过产品设计阶段，直接进入技术实现。

## 触发指令

```
/flow-quick [需求描述]
```

## 重要提示

**在执行此命令前，必须先读取完整的详细说明文件：**

`ai-coding/action-quick.md`

该文件包含完整的执行流程、详细步骤、输出格式等所有必要信息。请先读取该文件，然后按照其中的详细说明执行。

## 执行流程

1. 判断类型（Bug修复/小功能/技术优化/UI微调）
2. Q1: 快速分析
3. Q2: 直接实现
4. Q3: 验证
5. Git Commit

## 参数说明

- `[需求描述]`: 需求描述，用于判断类型和生成单号

## 适用场景

- Bug 修复
- 小功能迭代
- 技术优化
- UI 微调
