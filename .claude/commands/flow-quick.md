# ARC 轻量工作流

执行 ARC 轻量工作流（3阶段），用于 Bug 修复、小功能迭代、技术优化、UI 微调。

## 用户需求

$ARGUMENTS

## 执行流程

请严格按照 `ai-coding/action-quick.md` 定义的流程执行：

### 1. 初始化

1. 判断需求类型，生成对应单号：
   - Bug 修复 → `FIX_XXX`
   - 小功能 → `FEAT_XXX`
   - 技术优化 → `OPT_XXX`
   - UI 微调 → `UI_XXX`
2. 创建 Git 分支
3. 创建工作目录 `ai-coding/works/[单号]/`

### 2. 执行 3 阶段

| 阶段 | 目标 | 产出 |
|------|------|------|
| Q1 快速分析 | 5分钟内确认修改范围 | `quick-analysis.md` |
| Q2 直接实现 | 按清单逐一修改 | `changes.md` |
| Q3 验证 | 功能+编译+Lint+测试 | 验证结果 |

### 3. Q1 快速分析

1. 理解需求
2. 定位相关代码
3. 生成修改清单

### 4. Q2 直接实现

1. 按清单逐一修改
2. 每个文件修改后运行 `pnpm tsc --noEmit`

### 5. Q3 验证

- `pnpm tsc --noEmit` 无错误
- `pnpm lint` 无错误
- `pnpm test` 全部通过

### 6. 升级判断

如遇以下情况，建议升级到完整模式 (`/flow-start`)：
- 修改文件 > 10
- 需要新增数据表
- 需要新增 API > 3
- 需要新增页面

## 用户响应格式

- `确认` / `通过` - 进入下一步
- `修改 [内容]` - 修改指定内容
- `升级` - 升级到完整模式
- `取消` - 取消当前工作流

请开始执行工作流。
