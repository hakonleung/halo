# 重构工作流指令

纯技术重构，不改变功能行为，跳过产品设计阶段。

## 触发指令

```
/flow-refactor [范围] [目标]
```

**示例**: `/flow-refactor src/lib/auth-service.ts 拆分为多个小文件`

## 重要提示

**在执行此命令前，必须先读取完整的详细说明文件：**

`ai-coding/action-refactor.md`

该文件包含完整的执行流程、详细步骤、输出格式等所有必要信息。请先读取该文件，然后按照其中的详细说明执行。

## 执行流程

1. R1: 分析评估
2. R2: 安全网建立
3. R3: 逐步重构
4. R4: 验证确认
5. 合并提交

## 参数说明

- `[范围]`: 重构目标范围（文件或目录）
- `[目标]`: 重构目标描述

## 适用场景

- 性能优化
- 代码质量提升
- 架构调整
- 技术升级
- 债务清理
