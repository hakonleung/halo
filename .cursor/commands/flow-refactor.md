# 重构工作流

纯技术重构，不改变功能行为，跳过产品设计阶段。

## 触发指令

```
/flow-refactor [范围] [目标]
```

## 适用场景

- 性能优化
- 代码质量
- 架构调整
- 技术升级
- 债务清理

## 执行流程

1. **R1 分析评估** - 扫描代码，识别问题，制定计划
2. **R2 安全网建立** - 检查测试覆盖率，补充测试
3. **R3 逐步重构** - 每次只做一件事，每步验证和提交
4. **R4 验证确认** - 运行全量测试，对比基准

## 重构原则

- 行为不变
- 小步快跑
- 测试保障
- 可回滚
