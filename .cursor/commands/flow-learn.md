# 工作流学习指令

从归档的工作流中提取技术设计和技术经验，更新到项目配置文件中，确保后续 AI flow 可以吸取教训。

## 触发指令

```
/flow-learn
```

## 重要提示

**在执行此命令前，必须先读取完整的详细说明文件：**

`ai-coding/action-learn.md`

该文件包含完整的执行流程、详细步骤、输出格式等所有必要信息。请先读取该文件，然后按照其中的详细说明执行。

## 执行流程

1. 查找所有归档的 works，过滤未标记 learned 的归档
2. 读取 archive.md 和 overview.json
3. 提取技术设计和技术经验
4. 简化并分类内容
5. 更新 context/tech-config.md、.cursorrules、CLAUDE.md
6. 标记归档为 learned
7. 提交到 Git

## 参数说明

无需参数。

## 适用场景

- 归档的工作流中包含有价值的技术设计经验
- 需要将技术教训沉淀到项目配置中
- 确保后续开发能够避免重复错误或采用已验证的最佳实践
