# 用户响应格式

> 在工作流执行过程中，人类可使用的标准响应。

## 通用响应

- **`确认` / `通过`**: 确认当前内容，进入下一步
- **`修改 [具体内容]`**: 修改指定内容
- **`返工: [原因]`**: 当前阶段返工
- **`回退到 [阶段]: [原因]`**: 回退到指定阶段
- **`跳过`**: 跳过当前步骤（使用默认值）

## 特定场景响应

### 创建/初始化阶段

- **`重新开始`**: 重新开始当前阶段
- **`重新扫描`**: 重新扫描项目（仅 `/flow-init`）

### 轻量/迭代模式

- **`升级`**: 升级到完整模式
- **`取消`**: 取消当前工作流

### 重构模式

- **`跳过测试补充`**: 接受当前覆盖率，继续重构
