# project.md 询问清单

## 问题列表

1. **项目名称和定位**
   - 项目叫什么名字？ → `项目概述.名称`
   - 一句话描述项目定位？ → `项目概述.定位`

2. **目标用户**
   - 目标用户是谁？ → `项目概述.目标用户`

3. **核心功能**
   - 主要功能模块有哪些？ → `功能模块` (每个模块一个章节)
   - 每个模块的核心功能点？

4. **解决的痛点**
   - 这个项目要解决哪些痛点？ → `解决的痛点` (编号列表)

5. **页面结构**
   - 需要哪些页面？ → `页面结构` (表格)
   - 哪些需要登录才能访问？ → 标记为"公开"或"需登录"

6. **开发优先级**
   - P0 核心功能？ → `开发优先级.P0`
   - P1 重要功能？ → `开发优先级.P1`
   - P2 增强功能？ → `开发优先级.P2`
   - P3 优化项？ → `开发优先级.P3`

## 扫描提示 (仅 `/flow-init`)

- 项目名称：`package.json.name`
- 页面结构：扫描 `src/app/` 或路由文件
- 功能模块：根据页面和组件推断
