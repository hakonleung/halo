# Markdown 输出规范

> AI 在工作流中生成的所有 Markdown 文档必须遵循以下规范。

## 核心规则

### 1. 不使用 Emoji

- **禁止**：在文档标题、正文、列表项中使用任何 emoji 符号
- **示例错误**：`✅ 已完成`、`📝 注意事项`、`⚠️ 警告`
- **正确示例**：`已完成`、`注意事项`、`警告`

### 2. 不使用冗余样式

- **禁止**：过度使用粗体、斜体、代码块等样式标记
- **原则**：仅在必要时使用样式，保持文档简洁易读
- **示例错误**：`**重要**：**必须**在**所有**情况下**确保**...`
- **正确示例**：`重要：必须在所有情况下确保...`

### 3. 优先使用列表而非表格

- **原则**：尽量使用列表代替表格，除非数据确实需要表格形式展示
- **适用场景**：
  - 简单的键值对 → 使用列表
  - 步骤说明 → 使用有序列表
  - 检查项 → 使用无序列表
  - 复杂的数据对比 → 可保留表格

**示例：将表格转换为列表**

错误（表格）：
```markdown
| 属性 | 值 |
|------|-----|
| 名称 | NEO-LOG |
| 理念 | AI 原生 |
```

正确（列表）：
```markdown
- **名称**: NEO-LOG
- **理念**: AI 原生
```

## 样式使用指南

### 标题层级

- 使用标准的 `#` 到 `######` 标记
- 保持层级清晰，不要跳级（如 H1 后直接 H3）

### 列表格式

- 无序列表使用 `-`，有序列表使用数字
- 嵌套列表使用 2 个空格缩进
- 列表项内容简洁明了

### 代码块

- 仅在展示代码、配置、命令时使用代码块
- 指定语言类型：` ```typescript`、` ```bash` 等

### 强调文本

- **粗体**：仅用于关键术语、重要提示
- *斜体*：仅用于引用、术语定义
- `代码`：仅用于变量名、函数名、路径等

## 文档结构建议

1. 使用清晰的标题层级组织内容
2. 每个章节保持适当的长度（建议不超过 100 行）
3. 使用列表组织相关信息
4. 代码示例使用代码块
5. 重要提示使用引用块（`>`）

## 检查清单

生成 Markdown 文档后，检查以下项：

- [ ] 文档中无 emoji
- [ ] 无过度使用的样式标记
- [ ] 简单数据使用列表而非表格
- [ ] 标题层级清晰
- [ ] 代码块指定了语言类型
- [ ] 文档结构清晰易读
