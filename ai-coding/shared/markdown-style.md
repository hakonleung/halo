# Markdown 输出规范

> AI 在工作流中生成的所有 Markdown 文档必须遵循以下规范。

## 核心规则

1. **不使用 Emoji**：禁止在文档中使用任何 emoji 符号
2. **不使用冗余样式**：仅在必要时使用粗体、斜体等样式，保持文档简洁
3. **表格使用规则**：仅当列数大于 2 时使用表格，否则使用列表

## 表格使用规则

- **列数 ≤ 2**：使用列表
- **列数 > 2**：可使用表格

示例（2列，应使用列表）：
```markdown
- **名称**: NEO-LOG
- **理念**: AI 原生
```

示例（3列，可使用表格）：
```markdown
| 内容类型 | PRD 阶段 | 技术设计阶段 |
|---------|---------|------------|
| 用户故事 | 应该 | 不应该 |
```

## 样式使用

- **粗体**：仅用于关键术语、重要提示
- *斜体*：仅用于引用、术语定义
- `代码`：仅用于变量名、函数名、路径等
- 代码块：指定语言类型（` ```typescript`、` ```bash` 等）
